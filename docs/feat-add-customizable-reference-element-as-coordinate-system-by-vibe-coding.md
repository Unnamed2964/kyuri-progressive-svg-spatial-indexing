# feat: add customizable reference element as a coordinate system by vibe coding

日期：2026-04-26

说明：这份文档整理的是本次为索引控制器增加“可选参考元素坐标系”能力时的相关对话，不包含隐藏上下文或工具调用细节。

## 背景

用户首先追问了 SVG 缩放、CTM、world AABB 与父级坐标系之间的关系，核心目标是确认：

- 当前空间索引是否固定工作在 world 坐标系
- 如果业务更关心某个父容器 B 的局部坐标系，是否可以直接在 B 坐标系里建立索引
- 这项能力是否可以做成显式可选，而不是让调用方自己在外部再做一层坐标换算

这些讨论直接推动了本次特性：允许调用方指定一个参考元素 B，使控制器把每个被跟踪图元的 bbox 量到 B 的局部坐标系中。

## 相关对话摘录

### 1. 缩放是否会影响空间索引

用户：

> 请问 rmp 画布缩放时，会怎样

助手：

> rmp 不是去逐个改节点、线段的业务坐标，而是把可见内容放在一个带变换的 SVG viewport g 里，通过这个容器统一做缩放和平移。也就是说，节点本身的图数据通常还保持在“世界坐标”里，缩放只是显示层多了一层 viewport transform。

用户：

> 请问这会类似于空间索引整个重做吗

助手：

> 如果画布缩放改变了公共祖先变换，那么从索引内容上看，确实近似于“整个索引都需要重做一遍几何”；但在这个库里它不是自动重建整个索引实例，而应该显式做一次 invalidateAll()，然后 flush()，让所有条目批量重测更新。

### 2. 什么叫经过 CTM 变换后的 world AABB

用户：

> 请问什么叫“经过 CTM 变换后的 world AABB”

助手：

> 先拿元素在自己本地坐标系里的 bbox，然后用它的 CTM 把这个 bbox 的四个角变换到统一世界坐标里，最后再用这四个变换后的点重新包出一个坐标轴对齐矩形。这个结果就是 world AABB。

### 3. 能不能直接落到父元素 B 的坐标系

用户：

> 请问是否可以“A”在其父亲“B”的坐标系呢的坐标

助手：

> 可以。更准确地说，不是直接拿到“A 的坐标”这一个值，而是可以把 A 的几何从 A 自己的局部坐标系，换算到它父亲 B 的坐标系里。

助手：

> A 相对于 B 的变换矩阵是：CTM(B)^-1 · CTM(A)

### 4. 最终功能请求

用户：

> 希望改成可以选这个B坐标系

这个请求最终落成了控制器的新可选参数 `coordinateReferenceElement`。默认情况下，索引仍然工作在 SVG world 坐标系；如果传入参考元素 B，则：

- `bbox` 被测量到 B 的局部坐标系
- `queryRect(...)` 也需要使用 B 坐标系下的查询窗口
- 调用方不必再自己维护一套外部坐标换算逻辑

## 实现结果

本次实现包含以下内容：

- 在控制器选项中新增 `coordinateReferenceElement`
- 新增通用几何函数，用于把局部 bbox 测量到任意参考坐标系
- 默认行为保持兼容，未传参考元素时仍使用 world 坐标
- 补充测试覆盖“默认 world 坐标”和“指定参考元素 B 坐标系”两种场景
- 在 README 中新增“坐标系约定”说明
